Appendix B — The Golden Rule Templates
To be placed on tables for Phase 1

Each team must write two Golden Rules for the same everyday XR situation: one Technical Rule about how the system should work, and one Ethical Rule about what the system must protect or limit.

Start by naming the team's core tension: "between _____ and _____."
Then write both rules from that same tension so they support each other — the friction should appear later, when different teams' rules meet in the persona exercise.
Template A

Technical Rule

Focus on what the system must do to make the situation workable in practice.

“To make [DAILY ROUTINE CONTEXT] workable, any Co-Living system must [DESIGN REQUIREMENT], even if it means [PERFORMANCE / IMMERSION / CONVENIENCE TRADE-OFF].”
Design level · concrete enough to guide implementation · not code-level
Template B

Ethical Rule

Focus on what the system must protect, reveal, or constrain in that same situation.

“To protect [PERSON / VALUE / RIGHT] in [SAME DAILY ROUTINE CONTEXT], any Co-Living system must [SAFEGUARD OR LIMIT], even if it means [EFFICIENCY / PERSONALIZATION / SEAMLESSNESS TRADE-OFF].”
Human protection level · concrete enough to shape design · not abstract values only
Example pair · Threshold Engineers Technical: “To make cross-reality transitions in collaborative AR workable, any Co-Living system must preserve stable timing and cue coherence, even if it means slower or more explicit switching.”
Ethical: “To protect users from disorientation during those same transitions, any Co-Living system must keep reality shifts legible, even if it weakens perfect seamlessness.”
Example pair · Care Advocates Technical: “To make XR support workable in routine care, any Co-Living system must reduce onboarding and interaction friction, even if it means simpler interfaces.”
Ethical: “To protect patients from overload in that same care context, any Co-Living system must keep guidance adjustable to the user’s capacity and routine, even if it makes the system less standardized.”
Example pair · Cultural Sentinels Technical: “To make cultural XR experiences workable across contexts, any Co-Living system must preserve framing and provenance, even if it limits portability.”
Ethical: “To protect users from invisible memory or identity shaping in that same context, any Co-Living system must make mediation and influence visible, even if it reduces immersion.”
Reminder Do not try to merge everything from every paper. Choose one shared concern or one shared design principle that appears in more than one paper from your team, and build the pair from there.

Good pairs are clear, arguable, and slightly uncomfortable: they should make sense within your team, but create tension when applied to another team’s persona.
